# ProjectKoversunIndyk

Ця програма призначена для нотування тексту, його збереження на умовному сервері з кодуванням вмісту з можливістю перегляду, сортування та пошуку відповідної нотатки. Головна особливість програми у тому, що вона відображає затримку з якою кожна нотатка надсилається до умовного серверу та підраховує середню з усіх відправлених користувачем нотаток.
Функції, що виконує програма:
•	Запис нотаток.
•	Виведення на екран затримки тексту, що відправляється.
•	Підрахунок середньої затримки відправлених нотаток у профілі користувача.
•	Сортування по зростанню та зменшенню нотаток у профілі користувача за їх затримками.
•	Пошук нотатки у профілі користувача.
•	Збереження нотаток користувача у закодованому форматі, що робить їх неможливими для читання без ключа. Після її декодування на вікні інформації для праці з ними.
Середа розробки Visual Studio 2019, щонайменше Community версії. Повинна бути підтримка мови програмування C++/CLI та бібліотек Нлохманна та маршалювання, які вже знаходяться у файлах проєкту. Розуміння форматів файлів .csv, .json.

Розроблений проєкт складається з 7 модулей:
•	MyForm – цей модуль необхідний для того, щоб користувач програми пройшов автентифікацію.
•	Main – цей модуль необхідний для того, щоб користувач програми мав спроможність надсилати нотатки до профілю та бачити затримку поточного тексту.
•	Info – цей модуль необхідний для того, щоб користувач програми міг працювати з надісланими нотатками, шукати, сортувати, бачити середню затримку усіх нотаток у профілі.
•	Developers – цей модуль містить у собі інформацію про розробників програми.
•	Functions – цей модуль призначений для зберігання усіх авторських алгоритмів та функцій проєкту для їх використовування в інших модулях.
•	Files – цей модуль призначений для зберігання методів роботи з файлами у проєкті, їх відкриття, створення та змінення.
•	Exceptions – цей модуль призначений для обробки виняткових ситуацій, що виникають у різних місцях проєкту та виведення їх користувачу.

Тут представлені модулі, що відносяться до форм програми:
Модуль MyForm:
•	button1_Click – перевіряє наявність тексту у полях тексту та правильність його введення, створює або редагує файл з історією користувачів, відкриває наступну форму програми.
•	button2_Click – завершує роботу програми.
Модуль Main:
•	button1_Click – завершує роботу програми.
•	button2_Click – створює або відкриває профіль користувача, надсилає до нього введену нотатку.
•	button3_Click – видаляє текст з поля повідомлення та його затримку.
•	textBox1_TextChanged – підраховує значення затримки поточного тексту та виводить його на лічильник.
•	button4_Click – відкриває інформацію про розробників.
•	Main_Load – відкриває форму інформації одразу під час запуску цього модуля, відправляє туди ім’я файлу та дані про користувача.
Модуль Info:
•	button1_Click – оновлює список відправлених нотаток та підраховує їх середню затримку.
•	button2_Click – завершує роботу програми.
•	button3_Click – виконує пошук серед нотаток у профілі.
•	button4_Click – сортує нотатки на формі за зростанням або за зменшенням, порядок нотаток у файлі не змінюється.
•	Info_Load – виводить дані про користувача на форму.
Модуль Developers:
•	button2_Click – завершує роботу програми.

Структура програми

Програма складається з чотирьох форм й належних до них класів форм. У класах цих форм описана взаємодія користувача з інтерфейсом програми.
Усі функції та алгоритми, необхідні для підтримки та роботи цієї взаємодії, зберігаються у трьох авторських класах програми, що підключені до всіх класів форм: клас функцій, який містить такі алгоритми як кодування, декодування, пошук та сортування; клас файлів, у якому знаходяться основні методи по зміненню, відкриттю та створенню файлів; клас виняткових ситуацій, що передбачає та інформує користувача про можливі помилки при роботі з програмою.

Файли з форматом .h, а саме MyForm, Main, Info, Functions, Files, Exceptions та Developers, – це файли заголовків, вони зберігають у собі оголошення методів.
Файли з форматом .cpp, а саме MyForm, Main, Info, Functions, Files, Exceptions та Developers, – це файли з реалізацією оголошених у файлах заголовків методів.
Файли з форматом .resx, а саме MyForm, Main, Info та Developers, – це файли, що зберігають ресурси проєкта, наприклад зображення та його налаштування.
Файли з форматом .png, а саме refresh, info, circle та background, – це файли зображень, які необхідні для візуальної складової проєкту.
Файли .gitignore та .gitattributes зберігають інформацію, необхідну для правильного завантаження та роботи проєкту з GitHub.
Файли Project1.vcxproj та Project1.vcxproj.filters – файли проєкту. У них зберігається інформація, специфічна для кожного проєкту, а також інформація, у яку директорію додавати нові файли проєкту.
Файл packages.config необхідний для того, щоб зберігати налаштування додаткових пакетів проєкту.
Файл usershistory.json – це файл для зберігання інформації про усіх користувачів програми.
Файли формату .csv, наприклад ivanvan та petrovrov – це файли, що зберігають закодовані нотатки користувачів та затримки цих нотаток.
Файл Project1.sln – це головний файл проєкту, завдяки якому можна організувати усі інші файли проєкту в одне рішення.

Звертання до програми

Для того, щоб запустити програму, необхідно:
1.	Зайти на GitHub, де зберігається проєкт.
(github.com/ibuypower22/ProjectKoversunIndyk)
2.	Завантажити архів з програмою.
(ProjectKoversunIndyk-master.zip.)
3.	Розархівувати архів.
4.	Відкрити розархівовану кореневу папку програми під назвою ProjectKoversunIndyk-master.
5.	У цієй папці необхідно знайти та запустити файл проєкту під назвою Project1.sln.
6.	Відкриється проєкт Visual Studio з кодом програми.
7.	Для компілювання, програмісту необхідно натиснути на кнопку «Локальный отладчик Windows».

Вхідні дані

Ім’я, прізвище та логін користувача, текст надісланої або шуканої нотатки. Опціонально (тільки якщо цей користувач вже зберігав нотатки у програмі) .csv файл, що зберігає інформацію про нотатки одного користувача.

Вихідні дані

Файл формату .csv, що зберігає інформацію про нотатки одного користувача та файл формату .json, що зберігає історію усіх користувачів, що користувалися програмою.
